# 주차별 산출물 체크리스트

매주 교수자가 배포 전 확인할 파일은 동일합니다.

| 파일 | 학생 용도 | 교수자 점검 |
|---|---|---|
| `README.md` | 학습목표·수업 흐름 | 시간과 산출물 일치 |
| `practice.ipynb` | Colab 실습 | 전체 실행, 경로, assert 통과 |
| `slides.md` | 그림+단계별 실습 수업 | Marp front matter, 15장 구성, SVG 경로·노트북 코드 일치 |
| `assignment.md` | 과제 수행·제출 | 요구사항과 10점 루브릭 일치 |
| `quiz.md` | 형성평가 | 정답 키와 문항 번호 일치 |

## 학기 시작 전

- [ ] 15개 Colab 배지가 올바른 저장소 경로를 가리킨다.
- [ ] `datasets/`의 CSV 4종이 공개 가능한 가상 데이터인지 확인한다.
- [ ] LMS에 평가 비율과 제출 권한 안내를 게시한다.
- [ ] 학생에게 API 키·개인정보 업로드 금지를 안내한다.

## 매주

- [ ] GitHub의 최신 노트북을 Colab에서 다시 시작 후 전체 실행한다.
- [ ] 과제 제출 링크와 마감 시각을 게시한다.
- [ ] 대표 오류 1~2개를 익명화해 다음 수업 피드백에 사용한다.

## 8·12·14·15주

- [ ] 중간/최종 마일스톤 루브릭을 수업 전에 공개한다.
- [ ] 프로젝트 데이터 접근성과 재현성을 점검한다.
- [ ] 동료 피드백이 인신평가가 아닌 실행·근거 중심인지 확인한다.
