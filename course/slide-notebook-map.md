# 슬라이드–Colab 수업 연계표

15개 주차는 모두 같은 슬라이드 구조와 노트북 구조를 사용합니다. 교수자는 슬라이드를 설명한 직후 대응하는 Colab 셀을 학생과 함께 실행할 수 있습니다.

| 수업 단계 | 슬라이드 | Colab 노트북 | 교수자 진행 | 학생 증거 |
|---|---:|---|---|---|
| 도입 | 1~4장 | 제목·AI 기록 셀 | 목표·산출물·개념 안내 | 오늘의 목표 확인 |
| 개념 시각화 | 5장 | 단계별 실습 지도 | 4단계 순서도를 왼쪽부터 설명 | 코드 흐름 구두 설명 |
| Level 1 따라하기 | 6~8장 | `Level 1` 마크다운+코드 | 실행 전 출력 질문 | 예상·실제·변경 이유 |
| Level 2 바꾸기 | 9~10장 | `Level 2` 마크다운+코드 | 입력·처리·출력 표시 | 두 곳 변경 비교 |
| Level 3 적용하기 | 11~12장 | `Level 3` 마크다운+코드 | 정상·경계 사례 시범 | 테스트·오류·수정 기록 |
| Level 4 확장하기 | 13장 | 확장 코드+설계 메모 | AI 질문을 단계화 | 기능·채택/거절·검증 |
| 정리 | 14~15장 | 제출 점검·성찰 | 전체 실행과 제출 확인 | 체크리스트·3문장 성찰 |

## 수업 중 전환 문장

- 5장: “그림의 네 단계를 따라 오늘 코드가 처리되는 순서를 설명해 봅시다.”
- 6장: “실행 버튼을 누르기 전에 결과부터 예상해 봅시다.”
- 9장: “이제 입력·처리·출력 세 부분을 표시하고 두 곳을 바꿉니다.”
- 11장: “코드가 실행되는지와 결과가 정확한지를 서로 다른 테스트로 확인합니다.”
- 13장: “AI에는 전체 정답이 아니라 변경 위치, 이유, 테스트를 먼저 요청합니다.”
- 14장: “마지막으로 런타임을 다시 시작하고 모든 셀을 순서대로 실행합니다.”

## 배포 전 자동·수동 확인

- `python scripts/validate_package.py --execute`로 15개 노트북을 실행합니다.
- 각 `slides.md`가 15장이고 개념 SVG와 Python 코드 블록 세 개 이상을 포함하는지 확인합니다.
- 슬라이드의 Level 1·2·3 코드가 같은 주차 노트북에 포함되어 있는지 확인합니다.
