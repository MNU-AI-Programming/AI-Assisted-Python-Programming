# 교수자 운영 가이드

## 첫 수업 전

- GitHub 저장소와 Colab 배지를 학생 계정에서 열어 봅니다.
- 제출 예시를 하나 실행해 `프롬프트 → AI 제안 → 직접 수정 → 테스트`를 보여 줍니다.
- AI 사용은 허용하되 비밀번호, API 키, 실제 학생 개인정보 입력은 금지합니다.

## 매주 운영

1. 5분 진단: 지난 주 개념을 코드 한 줄로 확인합니다.
2. 15분 개념: 이번 주 핵심과 흔한 오류를 설명합니다.
3. 45분 안내 실습: 예측 후 실행, 입력 변경, assert 확인 순으로 진행합니다.
4. 35분 독립 실습: 학생이 기능 두 가지를 개인화합니다.
5. 10분 오류 갤러리: 익명 사례로 원인과 검증법을 공유합니다.
6. 10분 퀴즈·제출 점검: 링크 권한과 전체 실행을 확인합니다.

## 피드백 문장 틀

- `기능은 실행되지만 ___ 입력에서 결과가 달라집니다. 이 사례를 테스트에 추가하세요.`
- `AI 제안을 채택한 이유는 보이지만 직접 수정한 근거가 없습니다. 변경 전후를 한 줄씩 설명하세요.`
- `차트는 만들었지만 분석 질문과 연결이 약합니다. 제목을 결론형 문장으로 바꾸고 근거 수치를 표시하세요.`

## 난이도 조절

- 기초 학생: 완성 예제를 실행하고 값·조건 두 곳을 바꾸게 합니다.
- 표준 학생: TODO 기능과 경계 사례를 스스로 설계하게 합니다.
- 심화 학생: 함수화, 테스트 추가, 시간·공간 효율 또는 시각화 대안을 비교하게 합니다.

## 결석·재수강 대체

`README → slides → practice → quiz → assignment` 순으로 수행하게 하고, 전체 실행 화면과 3문장 성찰을 제출하도록 합니다.
