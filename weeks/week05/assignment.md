# Week 05 과제 — 함수·모듈·테스트 가능한 코드

## 과제 목표

여행 경비 계산기를 세 개 이상의 함수로 분해하고 테스트를 작성한다.

## 권장 수행 순서

| 단계 | 해야 할 일 | 완료 증거 |
|---|---|---|
| Level 1 | 쉬운 예제를 실행하고 값 한 곳 변경 | 예상·실제 결과 |
| Level 2 | 핵심 예제에서 두 곳 변경 | 변경 전후 비교 |
| Level 3 | 정상·경계 사례로 도전 코드 검증 | 테스트 기록 |
| Level 4 | 새로운 기능 한 가지 추가 | 코드·AI 활용·수정 이유 |

## 필수 요구사항

1. `practice.ipynb`를 복사해 자신의 이름으로 저장합니다.
2. 기본 예제를 그대로 제출하지 않고 입력·조건·기능을 두 가지 이상 변경합니다.
3. 정상 사례와 경계 또는 오류 사례를 각각 한 번 이상 실행합니다.
4. 사용한 AI Prompt, 채택한 제안, 직접 수정한 부분과 이유를 기록합니다.
5. 노트북을 `런타임 다시 시작 후 모두 실행`하여 오류가 없는지 확인합니다.

## 제출물

- Colab 링크 또는 `.ipynb` 파일
- 실행 결과가 포함된 노트북
- 3문장 성찰: 이해한 점, 검증한 점, 다음 개선점

## 평가 기준(10점)

| 기준 | 점수 |
|---|---:|
| 요구 기능이 실행됨 | 4 |
| 입력·경계 사례 검증 | 2 |
| 코드 이해와 직접 수정 근거 | 2 |
| AI 활용 기록과 재현 가능한 제출 | 2 |
| 합계 | 10 |

## 3시간 수업 확장

동료 한 명의 노트북을 실행하고, `잘된 점 1개 + 재현되지 않는 점 또는 개선점 1개`를 이슈 형식으로 작성합니다.

## 제출 전 체크

- [ ] 공유 권한이 열려 있다.
- [ ] 처음부터 끝까지 실행된다.
- [ ] 개인정보·API 키가 포함되지 않았다.
- [ ] AI가 작성한 부분과 내가 바꾼 부분을 구분했다.
