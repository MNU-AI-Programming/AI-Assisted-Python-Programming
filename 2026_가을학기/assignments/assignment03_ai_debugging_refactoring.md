## 2. 과제 목적

이 과제는 실행되지 않거나 잘못된 결과를 만드는 AI 생성 코드를 분석하고, 오류 원인을 테스트로 확인한 뒤 안전하고 읽기 쉬운 코드로 개선하는 실기 과제입니다. AI에게 수정 코드를 한 번에 요청하는 것이 아니라, 학생이 오류를 재현하고 가설을 세우며 수정 결과를 검증하는 과정에 중점을 둡니다.

## 3. 문제 상황

다음 코드는 AI가 작성한 지출 분석 프로그램입니다. 코드에는 자료형, 누적 계산, 데이터 검증, 통계 계산과 결과 선택에 관한 여러 결함이 있습니다.

프로그램의 요구사항은 다음과 같습니다.

- 금액을 숫자로 변환할 수 있고 금액이 0 이상인 행만 분석합니다.
- 공백을 제거한 범주명이 비어 있지 않아야 합니다.
- 유효한 지출의 전체 합계와 평균을 계산합니다.
- 범주별 지출 합계를 계산합니다.
- 지출 합계가 가장 큰 범주를 찾습니다.
- 제외된 행의 위치와 제외 이유를 기록합니다.
- 유효한 데이터가 없어도 프로그램이 중단되지 않아야 합니다.

## 4. 결함이 있는 시작 코드

다음 코드를 Colab에 복사하여 **수정하지 않은 상태로 먼저 실행**합니다.

```python
records = [
    {"category": "식비", "amount": "12000"},
    {"category": "교통", "amount": "8000"},
    {"category": "식비 ", "amount": "15000"},
    {"category": "교통", "amount": "-3000"},
    {"category": "식비", "amount": "오류"},
    {"category": "   ", "amount": "5000"},
]


def analyze_expenses(records):
    total = 0
    category_totals = {}

    for row in records:
        amount = row["amount"]

        if amount < 0:
            continue

        category = row["category"]
        total =+ amount
        category_totals[category] = amount

    average = total / len(records)
    top_category = min(category_totals, key=category_totals.get)

    return {
        "total": total,
        "average": average,
        "category_totals": category_totals,
        "top_category": top_category,
    }


print(analyze_expenses(records))
```

## 5. 과제 수행 단계

### 단계 1. 오류 재현

- 시작 코드를 그대로 실행합니다.
- 발생한 오류 메시지 또는 잘못된 결과를 복사합니다.
- 오류가 발생한 줄과 그 이유에 대한 첫 번째 가설을 작성합니다.

### 단계 2. 결함 목록 작성

코드에서 서로 다른 결함을 **6개 이상** 찾습니다. 실행 오류뿐 아니라 실행은 되지만 결과를 잘못 만드는 논리 오류도 포함합니다.

| 번호 | 결함 위치 | 증상 | 원인 가설 | 확인 방법 |
|---:|---|---|---|---|
| 1 | 예: `amount < 0` | 자료형 오류 | 문자열과 정수 비교 | `type(amount)` 출력 |

### 단계 3. 작은 단위로 수정

- 한 번에 하나의 결함만 수정합니다.
- 수정할 때마다 관련 테스트를 다시 실행합니다.
- 변경 전 코드, 변경 후 코드와 수정 이유를 기록합니다.
- 오류 메시지만 사라지는 것이 아니라 계산 결과가 정확한지 확인합니다.

### 단계 5. 자동 테스트 작성

`assert`를 사용하여 다음 테스트를 모두 작성합니다.

1. 정상적인 숫자 금액 두 건의 합계와 평균
2. 문자열 숫자 금액이 올바르게 변환되는 경우
3. 같은 범주의 금액이 누적되는 경우
4. 음수 금액이 제외되는 경우
5. 숫자로 변환할 수 없는 금액이 제외되는 경우
6. 범주가 공백인 행이 제외되는 경우
7. 빈 목록을 입력해도 오류가 발생하지 않는 경우

예시:

```python
result = analyze_expenses([
    {"category": "식비", "amount": "1000"},
    {"category": "식비", "amount": "2000"},
])

assert result["total"] == 3000
assert result["category_totals"]["식비"] == 3000
```

## 7. AI 활용 및 검증 기록

AI를 사용할 수 있지만 다음 순서를 지켜야 합니다.

1. AI에 코드를 보내기 전에 학생이 먼저 결함 세 가지 이상을 찾습니다.

3. AI가 찾지 못한 결함 또는 잘못 설명한 내용을 한 가지 이상 기록합니다.
4. AI의 수정안을 바로 채택하지 않고 `assert` 테스트로 확인합니다.
